# /maintenance 命令

定期维护：检查文档与代码的一致性，清理过期内容。

## 执行

依次运行以下检查，汇总报告：

### 1. 运行完整文档健康检查

```bash
python -m harness.lint.check_docs
```

重点关注：
- `docs/spec-code-drift` — 规格文档中提到的类在代码中不存在（文档腐烂信号）
- `docs/missing-test-file` — 模块缺少场景测试
- `docs/missing-fixtures` — 模块缺少测试数据

### 2. 检查已完成规格与实际代码的偏差

逐个扫描 `docs/plans/completed/` 中的规格文档：
1. 读取规格中的接口定义章节
2. 读取对应模块的 `interfaces.py`
3. 对比方法签名是否一致（方法名、参数、返回类型）
4. 如有偏差，列出具体差异并建议更新规格或代码

### 3. 检查 CLAUDE.md 索引是否过期

对照项目实际文件，检查 CLAUDE.md 中引用的路径和命令是否仍然有效。

### 4. 输出维护报告

汇总所有发现，按以下分类输出：
- **需修复**：文档与代码不一致、链接失效
- **需补充**：缺少测试文件、fixtures、规格文档
- **建议清理**：过期的规格、未使用的公共类型

对每个问题给出具体的修复建议。如果问题可以自动修复（如更新链接、补充 __init__.py），直接修复。
